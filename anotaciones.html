Anotaciones de Spring Boot

Configuración General

@SpringBootApplication: Indica un punto de entrada para la aplicación Spring Boot. Combina @Configuration, @EnableAutoConfiguration y @ComponentScan.

@Configuration: Indica que una clase declara uno o más métodos @Bean y que los contenedores de Spring deben procesar estos métodos para generar nuevas bean definitions.

Componentes

@Component: Indica que una clase es un componente Spring y será instanciada automáticamente como un bean.

@Service: Indica que una clase es un servicio y debe ser instanciada automáticamente como un bean.

@Repository: Indica que una clase es un repositorio y su objetivo es encapsular el conjunto de objetos almacenados en una base de datos.

Controladores y Respuestas HTTP

@RestController: Usada en clases para indicar que son controladores de Spring MVC que devuelven datos directamente al cuerpo de la respuesta.

@RequestMapping: Asigna las solicitudes HTTP a métodos de los controladores en Spring MVC.

@GetMapping, @PostMapping, @PutMapping, @DeleteMapping: Variaciones de @RequestMapping para diferentes tipos de métodos HTTP.

Inyección de Dependencia

@Autowired: Inyecta automáticamente un bean en un campo, constructor o método.

@Qualifier: Se utiliza para desambiguar múltiples beans de un mismo tipo durante la inyección de dependencias.

@Value: Permite inyectar valores desde archivos de propiedades en campos o métodos.

Otras Anotaciones Útiles

@PathVariable: Utilizado en métodos de controlador para extraer valores de variables de la URI.

@RequestParam: Utilizado en métodos de controlador para extraer parámetros de la cadena de consulta de la URI.

@RequestBody: Usada en métodos de un controlador para indicar que un parámetro debe estar vinculado al cuerpo de la solicitud HTTP.

@ControllerAdvice: Utilizada para definir clases de asesoramiento a nivel global para el manejo de excepciones y el enlace de datos.

@ExceptionHandler: Utilizado dentro de clases anotadas con @Controller o @ControllerAdvice para definir métodos de manejo de excepciones.

@Scheduled: Utilizada para indicar que un método debe ser ejecutado en intervalos de tiempo o en un momento específico.

@Profile: Indica que un componente o una configuración deben estar activos solo en un perfil específico.

@Primary: Indica que un bean debe tener prioridad cuando hay múltiples beans del mismo tipo.

@Lazy: Indica que un bean se inicializará solo cuando se haga referencia a él por primera vez.
